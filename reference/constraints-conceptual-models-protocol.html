Constraints & Conceptual Models Protocol

Prevent the error before it forms; teach the true model through the system image. Don't patch one gap with the other's tool.

The two models

How it actually worksSystem modelThe real behavior of the product.
What the user believesMental modelInferred entirely from the system image — the visible surface you control.

Match → predictable. Diverge → systematic, repeating errors. You can only ever change the system image.

The four constraints

TypeHow it limits the actionUI example
PhysicalThe form makes the wrong input impossibleSlider bounded 0–100; number-only field; picker that can't express "Feb 31"
LogicalThe user reasons the valid move from the situation"Continue" disabled until a choice is made; step 3 locked until step 2 done
SemanticThe meaning of the thing narrows its useA trash-can icon means delete; an envelope means message
CulturalA learned convention shared by a groupRed = danger; the hamburger = menu; X closes

The five steps

Name the action and its valid set. What is the user doing, and what counts as a valid input or move?
Constraint check (before the error). Is an invalid action hard or impossible? If yes, which type carries it — physical, logical, semantic, cultural?
Model check (after the action). What does the user believe it does, and what does it actually do? Mark where the two diverge.
Locate the gap. Constraint gap = invalid actions possible or caught too late. Model gap = behavior surprises the user the same way every time.
Prescribe to the right gap. Constraint → add/strengthen it, prefer prevention over an error message. Model → fix the system image: labels, scope, structure, feedback.
Watch-outs: a constraint guides; a restriction blocks and reports late — prefer prevention to correction. Over-constraining cages experts: scaffold for novices, let the practiced opt out. Mental models are borrowed from other apps — honor conventions unless the payoff beats the cost of re-teaching. You can't out-confirm a wrong model; correct the system image instead.
