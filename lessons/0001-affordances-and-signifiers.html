Affordances & Signifiers

You already build good signifiers by instinct. This lesson adds the vocabulary and the framework behind them. The exercises don't tell you what's wrong; you operate each interface and judge it yourself.

01The distinction

An affordance is a relationship: what an object lets an agent do. A checkbox affords toggling; a canvas affords scrolling. An affordance exists whether or not anyone notices it — it is about possibility, and it lives in the system, invisibly. — Norman, jnd.org

A signifier is the perceivable cue that tells the user the affordance is there and where to act: the raised fill of a button, the underline on a link, a "↕ drag to reorder" handle, a blinking caret. A signifier that can't be perceived isn't doing its job. — Norman, "Signifiers, Not Affordances"

Affordances determine what actions are possible. Signifiers communicate where the action should take place. You need both.
— Don Norman, The Design of Everyday Things, 2013

The pair is useful because it splits a vague question ("is this usable?") into two you can answer separately: can the system do the thing (affordance), and does the user know where to act (signifier)? Many interaction problems come down to a failure of just one of them.

02The confusion to unlearn

A common confusion: designers say "this button has great affordance" when they mean it has a great signifier. Norman made the same slip in 1988, when he wrote "perceived affordance," and later introduced "signifier" to correct it. — jnd.org The clickability is built into the system and invisible; the thing you're actually judging is the cue. Two failure modes follow from the distinction, and both recur through the lesson:

  • Hidden affordance — a real action with no signifier. It works; nobody knows to try.
  • False affordance — a signifier with no affordance behind it. It looks actionable; acting does nothing.

09The expert's nuance

Signifier strength should match action stakes

A primary, irreversible action (Delete account) deserves a loud signifier; a tertiary one (a "learn more" link) a quiet one. Equal-volume signifiers flatten hierarchy and make users hunt. Your visual-design eye is the asset — you already modulate emphasis; now you can justify it as signifier weighting.

Conventions are pre-installed signifiers

An underlined blue phrase, a hamburger, a trailing chevron — they work because users learned them elsewhere. Following convention means the signifier is already in the user's head; a novel one you must teach. Break convention only when the payoff beats the re-teaching cost. — nngroup.com

Feedback is a signifier too — for the next action

When a toggle slides and recolors on tap, that's feedback for the action just taken — and a signifier of the new state and what's now possible. (Deep dive in Lesson 02.)

10When a hidden affordance is fine

Where most "affordance audits" — and §02 taken literally — go wrong: treating every hidden affordance as a defect. That maximalist trap produces bloated, over-signified screens. Operate the file list below — hover or swipe a row and see what surfaces.

The rule that overrides "signify everything": no critical action may be reachable only by a hidden affordance. Hidden affordances are fine as accelerators — faster second paths for users who learned them — provided a discoverable primary path also exists.
One teammate: "put a visible Archive button on every row." Another: "leave it." Soundest position?
Archiving is common and important, so it can't live only behind a hidden gesture (rules out C). But a full button on every row destroys scannability (rules out B). The move: a discoverable primary path plus the swipe as a fast accelerator.
The principle that overrides the naïve one: the goal was never "signify every affordance." It's "guarantee a discoverable path to every action whose importance demands it, then let hidden affordances exist freely as accelerators." Two questions: (1) how critical is the action? and (2) is there a discoverable equivalent? Only "critical + no equivalent" is a true defect. The target word is discoverability, not total exposure.

Model answer: "Primary: dropping the button turns the whole-card click into a hidden affordance — and 'open details' is the card's critical action, so it can't be reachable only by an unsignified click. Keep a perceptible cue (chevron, hover state, or the persistent link). Swipe: 'Add to compare' is secondary, so a hidden gesture is fine as an accelerator — provided a discoverable equivalent exists elsewhere (in details, or a row menu). If it's the only path to compare, it's orphaned, and not acceptable." The three ideas to carry out of this: hidden affordance, critical-versus-accelerator, and discoverable equivalent.

13Strengthen the muscle

Turn the reasoning into a conscious procedure. Protocol → guided walk-through → real-world mission.

Part A — The protocol (run it on any element)

Name the affordance. What can this element actually do? Strip the styling away in your head.
Find the signifier. What perceptible cue communicates it? If you can't point to one, there isn't one.
Place it on the 2×2. Perceptible, hidden, false, or correct rejection?
If broken, branch. Hidden → is the action critical, and is there a discoverable equivalent? False → which interaction signifier is lying — pointer, hover-lift, underline?
Prescribe. Almost always: change perception, not function.
